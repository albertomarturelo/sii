# Test plan — F22 (Declaración Anual de Renta — estado + observaciones)

Live/manual plan for the F22 read surface, both surfaces. Wire contract:
[`../sii-contract/f22.md`](../sii-contract/f22.md) (ported from the Python `sii-cli`,
**not yet re-observed from the TS port** — this plan IS that re-validation).

**F22 is session-keyed (ADR-005):** it always reads the **session principal**, ignores
the operate pointer, and takes **no `--rut`**. Reading an empresa's F22 means logging in
**as the empresa** (logout→login). **PII-safe:** the código grid drops every
identity/bank código and F22 exposes no `raw` — verify that holds live.

> ⚠️ Production + your real renta. See [`README.md`](README.md) before running.

## Preconditions

1. On branch `feat/GH-19-f22-status`, `pnpm build` (so the CLI binary and
   `packages/mcp/dist/main.js` include the F22 surface).
2. **MCP only:** restart Claude Desktop / reload the `sii` MCP server; confirm
   `f22_status` is listed among the tools.
3. Be the **persona** (you, representante legal of ≥1 empresa) for the session-keyed
   case #C8/#M8. Note your own RUT as `SELF` and the empresa RUT as `EMPRESA`.
4. Pick a **`<AÑO_OK>`** you actually declared (e.g. last tax year) and a
   **`<AÑO_VACÍO>`** you did not (for the "sin declaración" case).

Capture from M5/M8/M6 (synthetic/redacted only here; keep real values out of git):

- **`vgte` form** — `"S"`/`"N"`, boolean, or `1`/`0`? (parser is tolerant; confirm.)

  `HEADER_CODIGOS` in `portal/f22.ts` with an `// observed …` citation.

  the observation date in the contract.
