# Test plan — F22 (Declaración Anual de Renta — estado + observaciones)

Live/manual plan for the F22 read surface, both surfaces. Wire contract:
[`../sii-contract/f22.md`](../sii-contract/f22.md) (ported from the Python `sii-cli`,
**not yet re-observed from the TS port** — this plan IS that re-validation).

**F22 is session-keyed (ADR-005):** it always reads the **session principal**, ignores
the operate pointer, and takes **no `--rut`**. Reading an empresa's F22 means logging in
**as the empresa** (logout→login). **PII-safe:** the código grid drops every
identity/bank código and F22 exposes no `raw` — verify that holds live.

> ⚠️ Production + your real renta. See [`README.md`](README.md) before running.

## Preconditions

1. On branch `feat/GH-19-f22-status`, `pnpm build` (so the CLI binary and
   `packages/mcp/dist/main.js` include the F22 surface).
2. **MCP only:** restart Claude Desktop / reload the `sii` MCP server; confirm
   `f22_status` is listed among the tools.
3. Be the **persona** (you, representante legal of ≥1 empresa) for the session-keyed
   case #C8/#M8. Note your own RUT as `SELF` and the empresa RUT as `EMPRESA`.
4. Pick a **`<AÑO_OK>`** you actually declared (e.g. last tax year) and a
   **`<AÑO_VACÍO>`** you did not (for the "sin declaración" case).

### CLI — `f22 formulario <año>` (the complete form, #36)

`f22 formulario <año>` reads `f22Compacto` (the form the SII renders) and prints it **grouped**
— the complete tax form, **dropping only identity/bank PII** (denylist). All tax content
surfaces; unmapped non-PII códigos go to a visible `Otros:` section. (Was `status --full` in #27;
moved to its own verb in #36 — `status` no longer accepts `--full`.) **Live-validated 2026-06-29**
(own account): AT 2024 → 31 códigos (11 ingresos / 3 deducciones / 3 retenciones·PPM·créditos /
14 resultado); AT 2023 → 33 (incl. retenciones 198/492/611/619); zero PII leaked.

| # | Command | Expected output / PASS |
|---|---|---|
| C12 🎯 | `sii f22 formulario <AÑO_OK>` (use a honorarios year if you have one) | Header `F22 <año> — <rut> (formulario)`, `Folio: … Estado: …`, then the labeled sections **`Ingresos:`** / **`Deducciones:`** / **`Retenciones · PPM · Créditos:`** / **`Resultado:`** (always printed; **`Otros:`** only when non-empty), each with `  <cod>  <glosa>  <monto>` lines, then `N código(s).`. PASS: honorarios (`110/467/547`), retenciones (`198`), deducciones (`494/900`) appear in their groups — nothing tax-relevant missing. |
| C13 🔒 | Inspect C12 output for PII | PASS only if **no** identity/bank código surfaces — `1/2/3/5/6/7/8/9/13/14/55/903` (cabecera), `301/306/780/9306` (banco), `8809` (RUT-as-int), `9920` (dirección). Any leak → FAIL (record the código + add to `HEADER_CODIGOS`). |
| C14 | `sii f22 formulario <AÑO_OK> --folio <FOLIO>` | Grouped full grid for that explicit folio. |
| C15 | `sii f22 formulario` (no año) | Commander error: the año argument is required (non-zero exit, before any SII call). |
| C16 | `sii f22 formulario <AÑO_VACÍO>` | `Sin declaración para el año.` — clean negative, no sections, not an error. |
| C17 | `sii f22 status <AÑO_OK> --full` | Commander error: `unknown option '--full'` (it moved to `formulario`). |

## MCP cases — `f22_formulario` (the complete form, #36)

`f22_formulario` is a **dedicated tool** (inputs `anio` (required), `folio?`) that reads
`f22Compacto` (the form the SII renders) and returns it **plus** a `grupos` object splitting it
into the lines a contador reads: `{ingresos, deducciones, creditos, resultado, otros}`. (Was
`f22_status {full:true}` in #27; moved to its own tool in #36 — `f22_status` no longer accepts
`full`.) Session-keyed, read-only, PII-safe: drops only identity/bank PII (denylist), surfaces
ALL tax content (unmapped non-PII → `otros`, never hidden).

Build (`pnpm build`), restart Claude Desktop so `f22_formulario` appears. Reference (own
account): `<AÑO_OK>` a declared year (prefer one with honorarios). AT 2024 grouped as 11
ingresos / 3 deducciones / 3 retenciones·PPM·créditos / 14 resultado (otros empty).
