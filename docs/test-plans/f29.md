# Test plan — F29 (Declaración Mensual de IVA — Fase 1)

Live/manual plan for the F29 read surface, both surfaces, **Fase 1** (the robust SDI-JSON
verbs; no GWT-RPC). Wire contract: [`../sii-contract/f29.md`](../sii-contract/f29.md).
Decision: [ADR-013](../decisions/013-f29-phased-read-gwt-rpc-deferred.md).

**F29 is session-keyed (ADR-005):** it always reads the **session principal**, takes **no
`--rut`/`rut`**, and **rejects** a representing operate pointer **up front** with an actionable
"log in as the empresa" message. Reading an empresa's F29 means being logged in **as the
empresa** (logout→login). **PII posture:** no `raw`; identity PII (`listCodBase`, the PP29
`traza`) is never read; the estado `monto` IS surfaced as `total` (your own monthly figure) but
never reaches the audit log.

**Fase 1 scope reminder:** `f29 formulario` shows the **propuesta** (`fuente:"propuesta"`), SII's
suggestion — sparse, and propuesta-specific códigos without an observed glosa appear in `otros`.
The **exact presented form** (full balance incl. totals 538/89/91) is **Fase 2** (not in MCP yet).

> ⚠️ Production + your real IVA. See [`README.md`](README.md) before running.

## Preconditions

1. On branch `feature/GH-18-f29-read-surface`, `pnpm build` (so `packages/mcp/dist/main.js`
   includes the F29 tools).
2. **Restart Claude Desktop / reload the `sii` MCP server**; confirm `f29_formulario`,
   `f29_overview`, `f29_status` are listed among the tools.
3. Be logged in **as the contributor whose F29 you want** (these read the principal). If it is an
   empresa, log in as that empresa. Note your RUT as `SELF` (and, for the session-keyed case, an
   `EMPRESA` you represent).
4. Pick a **`<MES_OK>`** you actually filed (e.g. `2026-05`), a **`<MES_VACÍO>`** you did not, and
   a **`<AÑO>`** to sweep (e.g. `2026`).

## CLI cases (`sii f29 …`)

The `sii` binary is not on `$PATH`; run it as `node packages/cli/dist/main.js …` from the repo
root. Below, **`sii`** is shorthand for that (or `alias sii='node packages/cli/dist/main.js'`).
**Output is JSON by default** (the `@altumstack/sii-core` contract — pipe to `jq`); add **`--human`** for the
readable rendering. The `operating as:` header + diagnostics go to STDERR (human mode only), so
STDOUT stays pure JSON. Check `echo $?` for the exit code where noted.

| # | Command | Expected output / PASS |
|---|---|---|
| C1 | `sii auth login` | Browser opens, you type the Clave, lands off `zeusr.sii.cl`. PASS: login OK, Clave never on argv. |
| C2 | `sii auth status --human` | Prints your RUT + `operating as:` header. Record it as `SELF`. |
| **Overview — "lo que pagué por mes"** |||
| C3 🎯 | `sii f29 overview <AÑO> --human` | Header `F29 — <rut> (<AÑO>-01 → <AÑO>-12)`, then one line per month: `  <periodo>  <estado>  folio <n>  total <monto>`. Filed months show `Vigente` + a `total`; others `Sin declaración`. PASS. |
| C4 | `sii f29 overview <AÑO>-01 <AÑO>-06 --human` | Exactly 6 rows, chronological (`<AÑO>-01 → <AÑO>-06`). |
| C5 | `sii f29 overview <MES_OK> --human` | Single-month overview (desde=hasta=`<MES_OK>`): one row. |
| C6 | `sii f29 overview` (no args) `--human` | Defaults to the current calendar year to date (`<año actual>-01 → <mes actual>`). |
| C7 | `sii f29 overview <AÑO>` (JSON, default) `| jq '.meses[0]'` | Pipeable JSON: `{periodo, tieneDeclaracion, estado, folio, total}`. PASS: STDOUT is pure JSON (header on STDERR). |
| C8 | `sii f29 overview 2020-01 2026-12` | Validation error *"Rango demasiado amplio … (máximo 36)"*, **before** any SII call. Non-zero exit. |
| C9 | `sii f29 overview <AÑO>-06 <AÑO>-01` (inverted) | Validation error *"Rango inválido: 'desde' … posterior a 'hasta'"*, before any SII call. Non-zero exit. |
| **Formulario — desglose de la propuesta** |||
| C10 🎯 | `sii f29 formulario <MES_OK> --human` | Header `F29 <MES_OK> — <rut> (formulario · fuente: propuesta)`, then non-empty group sections `Débitos (ventas):` / `Créditos (compras):` / `Retenciones, PPM e impuesto único:` / `Otros impuestos y ajustes:` / `Determinación y total a pagar:` with `  <cod> <signo>  <glosa>  <monto>` lines, then `N código(s).`. PASS. |
| C11 🔒 | Inspect C10 output for PII | PASS only if **no** identity/bank value appears (no nombre/RUT/dirección/cuenta; no `raw`). Any leak → FAIL (record it). |
| C12 | `sii f29 formulario <MES_VACÍO> --human` | `Sin propuesta para el período.` — clean negative, no sections, not an error. |
| **Status — estado del mes** |||
| C13 | `sii f29 status <MES_OK> --human` | Header `F29 <MES_OK> — <rut> (estado)`, then `  <fecha>  <estado>  folio <n>  total <monto>` per declaración, then `N declaración(es).`. PASS. |
| C14 | `sii f29 status <MES_VACÍO> --human` | `Nada presentado para el período.` — clean negative, not an error. |
| **Cross-cutting** |||
| C15 🔑 | `sii operate <EMPRESA>` then `sii f29 status <MES_OK>` | **Session-keyed proof:** **rejected** with *"El F29 es session-keyed… inicia sesión como ella (logout→login)"* echoing the **empresa RUT** (NOT the razón social). Non-zero exit, **no SII call**. |
| C16 | `sii operate --self` then `sii f29 status <MES_OK>` | Now succeeds, reads `SELF`. |
| C17 | `sii f29 status 2026-13` / `sii f29 formulario abc` | Validation error *"Período inválido…"* / mes 1–12, before any SII call. Non-zero exit. |
| C18 | `sii auth logout` then `sii f29 status <MES_OK>` | `NotAuthenticated` (actionable re-login hint), **exit code 2**. |
| C19 | `sii f29 status <MES_OK>` (JSON, default) `| jq '.error // .periodo'` | In JSON mode an error surfaces as `{"error":"<verbatim>"}`; success as the result object. PASS: STDOUT parseable. |
