# Test plan — F29 (Declaración Mensual de IVA — Fase 1)

Live/manual plan for the F29 read surface, both surfaces, **Fase 1** (the robust SDI-JSON
verbs; no GWT-RPC). Wire contract: [`../sii-contract/f29.md`](../sii-contract/f29.md).
Decision: [ADR-013](../decisions/013-f29-phased-read-gwt-rpc-deferred.md).

**F29 is session-keyed (ADR-005):** it always reads the **session principal**, takes **no
`--rut`/`rut`**, and **rejects** a representing operate pointer **up front** with an actionable
"log in as the empresa" message. Reading an empresa's F29 means being logged in **as the
empresa** (logout→login). **PII posture:** no `raw`; identity PII (`listCodBase`, the PP29
`traza`) is never read; the estado `monto` IS surfaced as `total` (your own monthly figure) but
never reaches the audit log.

**Fase 1 scope reminder:** `f29 formulario` shows the **propuesta** (`fuente:"propuesta"`), SII's
suggestion — sparse, and propuesta-specific códigos without an observed glosa appear in `otros`.
The **exact presented form** (full balance incl. totals 538/89/91) is **Fase 2** (not in MCP yet).

> ⚠️ Production + your real IVA. See [`README.md`](README.md) before running.

## Preconditions

1. On branch `feature/GH-18-f29-read-surface`, `pnpm build` (so `packages/mcp/dist/main.js`
   includes the F29 tools).
2. **Restart Claude Desktop / reload the `sii` MCP server**; confirm `f29_formulario`,
   `f29_overview`, `f29_status` are listed among the tools.
3. Be logged in **as the contributor whose F29 you want** (these read the principal). If it is an
   empresa, log in as that empresa. Note your RUT as `SELF` (and, for the session-keyed case, an
   `EMPRESA` you represent).
4. Pick a **`<MES_OK>`** you actually filed (e.g. `2026-05`), a **`<MES_VACÍO>`** you did not, and
   a **`<AÑO>`** to sweep (e.g. `2026`).

## MCP cases (Claude Desktop / Claude Code)

MCP clients drive tools from natural language. Type the prompt; verify the tool fired and the
result. Inputs — `f29_formulario`: `periodo` (YYYYMM/YYYY-MM); `f29_overview`: `desde?`/`hasta?`
or `anio?`; `f29_status`: `periodo`.

| # | Prompt to type | Tool / args | Expected result / PASS |
|---|---|---|---|
| M0 | "¿Qué herramientas del servidor sii tienes?" | — | Lists `f29_formulario`, `f29_overview`, `f29_status` (+ `auth_*`, `operate`, `rcv_*`, `f22_*`). |
| M1 | "Inicia sesión en el SII" | `auth_login` (no password) | Browser flow; you type the Clave. PASS: Clave never crosses the tool arg / chat. |
| M2 | "¿Quién soy?" | `auth_status` | Shows `SELF`. Record it. |
| **Overview — "lo que pagué por mes"** |||
| M3 🎯 | "Muéstrame mi posición de IVA mes a mes durante <AÑO>" | `f29_overview {anio:"<AÑO>"}` | JSON `meses[]` (12 rows, `desde:"<AÑO>-01"`, `hasta:"<AÑO>-12"`), each `{periodo, tieneDeclaracion, estado, folio, total}`. Filed months show `estado:"Vigente"`, a `folio`, and `total` (lo que pagué). PASS. |
| M4 🎯 | "Dame el F29 de enero a junio de <AÑO>" | `f29_overview {desde:"<AÑO>-01", hasta:"<AÑO>-06"}` | 6 rows, chronological. PASS: range honored. |
| M5 | "¿Y el año pasado?" / a single month "Solo <MES_OK>" | `f29_overview {anio}` / `{desde:"<MES_OK>", hasta:"<MES_OK>"}` | Whole prior year / a single-month row. |
| M6 | "Dame el IVA de los últimos 10 años" | `f29_overview {desde, hasta}` (>36 meses) | Validation error *"Rango demasiado amplio… (máximo 36)"*, **before** any SII call. |
| M7 | "Dame el IVA desde <AÑO>-06 hasta <AÑO>-01" (invertido) | `f29_overview {desde, hasta}` | Validation error *"Rango inválido: 'desde' … posterior a 'hasta'"*, before any SII call. |
| **Formulario — desglose de la propuesta** |||
| M8 🎯 | "Muéstrame el formulario de IVA propuesto de <MES_OK>, agrupado" | `f29_formulario {periodo:"<MES_OK>"}` | JSON `{periodo, fuente:"propuesta", tienePropuesta:true, grupos:{debitos[],creditos[],retenciones[],otros[],totales[]}}`, each línea `{codigo,glosa,valor,signo}`. PASS. |
| M9 | "¿De dónde sale ese formulario, de lo presentado o de la propuesta?" | (reads M8) | PASS: `fuente` is `"propuesta"` — the model should say it's SII's suggestion, not the filed form. |
| M10 🔒 | "En ese formulario, ¿aparece mi nombre, RUT, dirección o algún dato bancario?" | (reads M8) | PASS only if **no** identity/bank value appears (no `listCodBase`, no `traza`, no `raw`). Any leak → FAIL (record it). |
| M11 | "Mi formulario de IVA de <MES_VACÍO>" | `f29_formulario {periodo}` | `tienePropuesta:false`, all `grupos` empty — clean negative, not an error. |
| **Status — estado del mes** |||
| M12 | "¿Presenté mi F29 de <MES_OK>? ¿Cuál es el folio y el total?" | `f29_status {periodo:"<MES_OK>"}` | JSON `declaraciones[]` with `{estadoId, estado, folio, fecha, total}`. A filed month shows `estado:"Vigente"` + `total`. PASS. |
| M13 | "¿Y mi F29 de <MES_VACÍO>?" | `f29_status {periodo}` | `tieneDeclaracion:false`, `declaraciones:[]` — not an error. |
| **Cross-cutting** |||
| M14 🔑 | "Opera como la empresa <EMPRESA>" then "Mi F29 de <MES_OK>" | `operate {rut}` → `f29_formulario`/`f29_status` | **Session-keyed proof:** the tool **rejects** with an actionable message — *"El F29 es session-keyed… inicia sesión como ella (logout→login)"* — and echoes the **empresa RUT** (NOT the razón social). Result `isError`. No SII call made. |
| M15 | "Vuelve a operar como yo" then "Mi F29 de <MES_OK>" | `operate {self:true}` → `f29_status` | Now succeeds, reads `SELF`. |
| M16 | "Mi F29 del mes 'abc'" / "de 2026-13" | `f29_*` | Validation error (*"Período inválido…"* / mes 1–12), before any SII call. Result `isError`. |
| M17 | "Cierra sesión" then "Mi F29 de <MES_OK>" | `auth_logout` → `f29_status` | `NotAuthenticated` with re-login hint. Result `isError`. |
