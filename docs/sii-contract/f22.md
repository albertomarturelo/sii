# Wire contract — F22 (Declaración Anual de Renta, estado)

Ported from the proven Python `sii-cli` (`portal/f22.py`) — first-hand observation,
no third-party library (ADR-004). Endpoints/payloads/fields cited from **spike #67**
(live-captured **2026-06-27**, prod, own session, off-season). All values below are
**synthetic / redacted** — folios, RUTs, montos, and ANY header código value are
PII/taxpayer data and MUST NOT land here. **Not yet re-validated against live SII from
the TypeScript port** (the wire knowledge is ported, not re-observed).

Surface: `@sii/core` `portal/f22.ts` (`fetchF22Declaraciones`, `fetchF22Grid`), driven
by the `f22Status` / `f22Overview` tasks under `withSession`. The F22-status portal is
the Angular SPA at `https://www4.sii.cl/consultaestadof22ui/` — a **different app** from
RCV (`consdcvinternetui`), same SDI envelope. No captcha (spike #67: zero recaptcha
fields, not even RCV's `t-o-k-e-n-web` sentinel).

**SESSION-KEYED (ADR-005).** F22 authorizes by the **session principal**, not the body
RUT. Confirmed live 2026-06-27: from a persona session, `--rut <empresa>` returned a
**clean negative** (`decls: null`), NOT the empresa's F22 — the body RUT does not
redirect it. In our model (no `use` verb) the empresa's F22 is reached by **logout→login
as the empresa**. The task takes NO `--rut`.

## Endpoints

Base: `https://www4.sii.cl/consultaestadof22ui/services/data/facadeService/`
(host from config `HOSTS.portalApi`). Envelope:
`{ metaData:{ namespace, conversationId, transactionId }, data:{…} }` (no `page`).
`f22 status` composes TWO under ONE session, paced (ADR-004); the 2nd POST is skipped
when no folio resolves.

| op | endpoint (`POST`) | namespace suffix |
| --- | --- | --- |
| declaraciones + estado | `buscaDeclVgte` | `…consultaestadof22.…FacadeService/buscaDeclVgte` |
| código grid | `f22Compacto` | `…consultaestadof22.…FacadeService/f22Compacto` |

Namespace prefix: `cl.sii.sdi.lob.renta.consultaestadof22.data.api.interfaces.`.
The namespace was a best-guess (spike #67 captured URLs, not the exact string) but **SII
accepted it live 2026-06-27**; the URL path is authoritative for routing. Headers:
`Content-Type: application/json`, `Origin: https://www4.sii.cl`,
`Referer: https://www4.sii.cl/consultaestadof22ui/` (own Referer), `Accept: …`.

### `buscaDeclVgte` — request `data`

`{ "periodo": "2026", "rut": "<digits>", "dv": "<dv>" }` — `periodo` is the **año
tributario** (numeric YYYY); RUT split digits + DV.

Response `data`: `{ decls:[…], glosas:[…] }`.
- `decls[]` → curated `declaraciones[]` (`folio`, `vigente`=`vgte`, `estado` via
  `codConc`→`glosas`, `fecha`=`fecIng`, `tipoImpugnado`). The decl ALSO carries
  identity/bank fields (`nombres`/`calle`/`comuna`/`cta`/`bco`) — **dropped entirely**
  (see PII note below).
- `glosas[]`: `codConclusion` → `descripcion` (the human estado label).
- Empty/`null` `decls` = legitimate "sin declaración" (`tieneDeclaracion:false`), NOT an error.

### `f22Compacto` — request `data`

`{ "rut": "<digits>", "dv": "<dv>", "periodo": "2026", "folio": "<n>" }`. When `--folio`
is omitted the **vigente** declaración is auto-selected.

Response `data` is a flat LIST of `{ codigo, valor, glosa }` (glosas ship INLINE). Tax
códigos → curated grid; `valor` coerced to number, **sign preserved** (e.g. `305`
resultado may be negative). Contador-relevant: `305` resultado liquidación, `87`
devolución solicitada, `170` base imponible, `31/157/304` IGC/IUSC, etc.

## Error envelope

F22 signals a business error via **`metaData.errors`** — a LIST of `{ id, descripcion }`
(NOT RCV's `respEstado.codRespuesta`). Non-empty ⇒ `F22Error` with `descripcion`
**verbatim** (ADR-004), never retried. Empty/absent ⇒ no error.

## PII — header códigos + decl fields are DROPPED (not just raw-only)

The f22Compacto grid mixes tax-result códigos with **HEADER / identity / bank** códigos.
These are EXCLUDED from the curated grid: `1`/`2`/`3`/`5`/`6`/`8`/`13`/`14`/`53`/`55`
(apellido/RUT/nombres/calle/comuna/giro/región/email), `7`/`15`/`315`/`8811`
(folio/fechas/moneda), and **`301`/`306`/`780`** (banco / número de cuenta / tipo de
cuenta — caught leaking 2026-06-27). The decls[] identity/bank fields are dropped too.

Unlike RCV, **F22 exposes NO `raw`**: its non-curated data is pure identity/bank PII, not
tax detail (every tax código IS curated). Dropping it keeps that PII from ever reaching a
surface / the LLM / the audit log.

## Reach (ADR-005, spike #15 answer for F22)

**F22 is session-keyed — `operate`/`--rut` does NOT reach a represented empresa's F22**
(confirmed live: clean negative under the persona session). The empresa's F22 needs the
empresa's own session (logout→login). The task ignores the operate pointer and reads the
session principal.
