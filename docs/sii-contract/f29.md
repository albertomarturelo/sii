# Wire contract — F29 (Declaración Mensual de IVA: propuesta + estado)

Ported from the proven Python `sii-cli` (`portal/f29.py` + its own `docs/sii-contract/f29.md`)
— first-hand observation, no third-party library (ADR-004). Endpoints/payloads/fields cited
from the Python live capture (**2026-06-12**, prod, own session). All values below are
**synthetic / redacted** — folios, RUTs, montos, names, and any `listCodBase` value are
PII/taxpayer data and MUST NOT land here. **Not yet live-revalidated from the TS port** (the
TS facade is built against the ported contract, like F22 status was before #27 revalidated it);
revalidate against a real session and refresh the date when an empresa/persona session is
available.

Surface: `@sii/core` `portal/f29.ts` (`fetchF29Propuesta`, `fetchF29Estado`), driven by the
`f29Draft` / `f29Status` tasks under `withSession`. The F29 portal is the Angular SPA at
`https://www4.sii.cl/propuestaf29ui/` — its own app (own Referer), same SDI envelope family as
RCV/F22. No captcha observed.

**SESSION-KEYED (ADR-005).** F29 authorizes by the **session principal**, not the body RUT.
Confirmed live in the Python project (**2026-06-26**): setting the body RUT to a represented
empresa returns `Consulta RUT[…] no esta autorizado`. In our model (no `use` verb) the empresa's
F29 is reached by **logout→login as the empresa**. The task takes NO `--rut` AND rejects a
representing operate pointer **up front** (before any session) with an actionable message —
rather than firing the doomed POST or silently reading self.

## Endpoints

Base: `https://www4.sii.cl/propuestaf29ui/services/data/facadeAdapterService/`
(host from config `HOSTS.portalApi`). Envelope: the shared SDI shape — `{ metaData, data }`.
Error channel is **`metaData.errors`** (a list of `{id, descripcion}`) — like F22, NOT RCV's
`respEstado`. `conversationId` = the session `TOKEN` cookie; `transactionId` = a per-call UUID.

| Facade | Purpose | `data` shape |
| --- | --- | --- |
| `getDeclaracionConCondicionesYTipoPropuesta` | IVA **propuesta** (SII's pre-filled F29 draft) | object (or `null` = sin propuesta) |
| `getDeclaracionConEstados` | **presented/saved** F29 records (estado) | array (`[]` = nada presentado) |

Namespaces (the URL path routes; the namespace is sent in `metaData`):
- `cl.sii.sdi.lob.iva.propuestaf29.data.api.interfaces.FacadeAdapterService/getDeclaracionConCondicionesYTipoPropuesta`
- `cl.sii.sdi.lob.iva.propuestaf29.data.api.interfaces.FacadeAdapterService/getDeclaracionConEstados`

## Request bodies (`data`)

The **período is split** into `mes` (zero-padded `MM`) + `anno` (`YYYY`) — the combined
`YYYYMM` form is NOT used here. The internal F29 form id is **`"2"`** (the form número is 29).
**SII names the same fields differently per facade** (its own inconsistency, observed):

Propuesta:
```jsonc
{ "rutContribuyente": "<digits>", "dv": "<DV>", "formCodigo": "2", "mes": "05", "anno": "2026" }
```
Estado (note `rut` not `rutContribuyente`, `formId` not `formCodigo`):
```jsonc
{ "rut": "<digits>", "dv": "<DV>", "formId": "2", "mes": "05", "anno": "2026" }
```

## Propuesta response (`data`, synthetic)

```jsonc
{
  "tipopropuesta": 40,             // propuesta-type id → curated `tipoPropuesta`
  "estado": 0,                     // propuesta-estado id → curated `estado`
  "descripcionEstado": null,
  "listCodPropuestos": [           // THE proposed F29 lines → curated `codigos`
    { "codigo": "511", "valor": "1097" },
    { "codigo": "538", "valor": "7482" },
    { "codigo": "115", "valor": "0.125" }   // a RATE (PPM tasa) — float, must survive
  ],
  "listCodAdministrativos": [      // 91xx control mirror → curated `codigosAdministrativos`
    { "codigo": "9114", "valor": "1097" }
  ],
  "listCodBase": [ /* PII — DROPPED */ ],    // header: nombre/RUT/dirección/comuna (códigos 01/02/03/05/06/08/15)
  "resultadoCalculoPP29": { "traza": "RUT[…] Periodo[…]" }  // traza EMBEDS the RUT — DROPPED
}
```

`valor` is numeric **without** thousands separators and with a `.` decimal point
(`"2709150"` → 2709150; `"0.125"` → 0.125) — so a plain `Number()` is correct here, **unlike F22**
(whose grid is es-CL-formatted). Floats are preserved (an int-only coercion would silently drop a
rate like `0.125` to null — caught in the Python live validation).

## Estado response (`data`, synthetic)

```jsonc
[
  {
    "estadoDeclaracionId": 1,          // → curated `estadoId` (read `estado` label instead)
    "estado": "Vigente",               // label: "Guardada" (saved draft) / "Vigente" (filed)
    "folio": 7654321,                  // presentation folio (0 when only saved, not filed)
    "declFechaCreacion": "12/06/2026", // verbatim DD/MM/YYYY → curated `fecha`
    "monto": 999999,                   // taxpayer financial position — DROPPED (PII-minimal)
    "enNegocio": false,
    "codigo": 0
  }
]
```
Empty `data: []` = nada presentado for the período (a clean negative, NOT an error).

## PII posture (ADR-004 / ADR-006)

F29 curates **only the tax códigos** (`listCodPropuestos` + `listCodAdministrativos`) and the
estado metadata, and exposes **NO `raw`** (like F22). The identity/financial PII is cleanly
**segregated into separate fields**, so — unlike F22 — **no per-código denylist is needed**; the
facade simply never reads them:
- `listCodBase` — the form header (nombre, RUT, dirección, comuna). Never read.
- `resultadoCalculoPP29.traza` — a debug string that **embeds the RUT**. Never read.
- estado `monto` — the taxpayer's financial position. Dropped on purpose (read IF/how it was
  filed, not the amount). Reconsider exposing it if a contador use-case needs it.

## Observed códigos (synthetic / illustrative)

- `listCodPropuestos` examples: `511`, `538`, `520`, `562`, `584`, `519` (persona);
  `142`, `586`, `504`, `151`, `563`, `115` (empresa). `115` = PPM **tasa** (a rate, e.g. `0.125`).
- `listCodAdministrativos`: the `91xx` family (e.g. `9114`).
- `listCodBase` (PII header — dropped): `01`, `02`, `03`, `05`, `06`, `08`, `15`.

## Quirks

- `formId` (estado) vs `formCodigo` (propuesta) — same internal id `"2"`, different key.
- `resultadoCalculoPP29.traza` embeds the RUT — never log/surface it.
- A `data: null` propuesta with no `metaData.errors` is "sin propuesta", not an error.
- No reCAPTCHA observed (the Python notes the `rb_*` POSTs are Dynatrace RUM beacons).

## Out of scope (write path)

The GWT-RPC editable-form layer (`rfiInternet/formularioFacade`, "Enviar Declaración") is the
F29 **write** path — out of scope here (needs its own legal ADR; ROADMAP "Write surfaces").
This contract covers the **read-only** propuesta + estado only.
